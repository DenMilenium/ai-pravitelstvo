## 🔧 ЧТО НУЖНО ДОДЕЛАТЬ / УЛУЧШИТЬ

### 1. 🔗 Связь агентов с GitHub
**Проблема:** Сейчас агенты работают в изоляции, код сохраняется в SQLite, но не пушится в GitHub автоматически.

**Решение:** Добавить автоматический push кода в GitHub после выполнения задачи.

### 2. 🔄 Двусторонняя синхронизация
**Проблема:** Только Dashboard → GitHub. Нет GitHub → Dashboard.

**Решение:** Добавить webhook для получения обновлений из GitHub.

### 3. 🎯 Назначение агентов на Issues
**Проблема:** При синхронизации с GitHub Issues не назначаются на конкретных агентов.

**Решение:** Добавить assignee при создании Issue.

### 4. 📊 GitHub Projects (новый v2)
**Проблема:** Используется старый API Projects.

**Решение:** Обновить на GraphQL API v2.

## 🎯 РАБОЧИЙ ПОТОК (Workflow)

### Сценарий 1: Создание проекта через Dashboard
```
1. Пользователь → Dashboard → Новый проект
2. TeamLead-Agent → Анализ ТЗ → Создание задач
3. Задачи сохраняются в SQLite
4. Пользователь → Выполнить задачу → TaskExecutor
5. TaskExecutor → Генерация кода → Сохранение артефактов
6. Пользователь → Собрать ZIP → ZIP архив
7. Пользователь → Скачать ZIP → Ручной деплой на Reg.ru
```

### Сценарий 2: GitHub интеграция
```
1. Пользователь → Синхронизировать с GitHub
2. GitHubClient → Создание Project в GitHub
3. Для каждой задачи → Создание Issue
4. Код из артефактов → push-file в репозиторий
```

### Сценарий 3: Чат с агентом
```
1. Пользователь → /agent/frontend
2. Сообщение → POST /agent/frontend/chat
3. Агент → Ответ (статус, задачи, выполнение)
4. "Выполни задачи" → TaskExecutor → Выполнение
```

## 🚀 РЕКОМЕНДАЦИИ

### Чтобы всё работало идеально:

1. **На сервере:**
   ```bash
   export GITHUB_TOKEN="your_token"
   export GITHUB_REPO="DenMilenium/ai-pravitelstvo"
   ```

2. **Для GitHub интеграции:**
   - Токен должен иметь права: `repo`, `project`, `workflow`
   - Репозиторий должен быть публичным или токен с доступом

3. **Для агентов:**
   - Все агенты должны наследоваться от `BaseAgent`
   - Реализовывать `process_task()`
   - Отправлять статус TeamLead через message bus
